Project Overview

This project represents the backend infrastructure for Siminds Virtual Labs. It is built as a Hybrid Cloud Architecture, leveraging the serverless scalability of Google Firebase (Gen 2 Cloud Functions) for application logic and the relational implementation of PostgreSQL for complex data relationships.

This hybrid approach allows the system to handle real-time user events while maintaining structured, query-heavy data for educational analytics.

Key Features

Strict Session Management

  • Concurrency: Custom logic in blocking functions to prevent account sharing.
  • Limits: Enforces max concurrent users per institution.

Scoring Engine

  • 5Es Model: Implements Engage, Explore, Explain, Elaborate, Evaluate model.
  • Auto-Grading: Automated grading of complex experiment results.

Architecture

  • Event-Driven: Listeners for Auth, Courses, and User events.
  • Hybrid DB: SQL for core relations + Firestore for realtime docs.

Tech Specs

  • Runtime: Node.js with TypeScript.
  • Compute: Firebase Cloud Functions Gen 2.

Performance at Scale

Gen 2 Concurrency

Handles 1000+ concurrent requests per instance, dramatically reducing cold starts compared to Gen 1.

Optimized Queries

Consolidated SQL joins reduce critical login path latency by ~66% compared to separate fetches.

Server-Side Computation

Complex scoring logic is processed server-side to ensure consistency and prevent client-side tampering.
